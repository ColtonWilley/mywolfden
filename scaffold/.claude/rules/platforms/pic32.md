# PIC32 / Harmony — wolfSSL Platform Guide

## 1. Overview

The Microchip PIC32 is a 32-bit MIPS-based microcontroller family used in embedded networking and IoT applications. wolfSSL supports the PIC32 platform through two primary defines:

- **`MICROCHIP_PIC32`** — General PIC32 platform support, originally targeting the Microchip PIC32 Ethernet Starter Kit.
- **`WOLFSSL_MICROCHIP_PIC32MZ`** — Enables support for the PIC32MZ Crypto Engine (hardware-accelerated cryptography available on PIC32MZ-class devices).

wolfSSL also supports the related **PIC24** family (16-bit) via MPLAB X project files using the XC16 compiler, documented under `IDE/MPLABX16/`.

Microchip Harmony is the framework commonly used with PIC32 devices. wolfSSL integrates with Harmony-based projects through MPLAB X IDE project files and a `user_settings.h` configuration approach.

---

## 2. Build Configuration

### Key Defines

| Define | Purpose |
|--------|---------|
| `MICROCHIP_PIC32` | Enables general PIC32 platform support |
| `WOLFSSL_MICROCHIP_PIC32MZ` | Enables PIC32MZ Crypto Engine hardware acceleration; also automatically defines `MICROCHIP_PIC32` if not already set |
| `MICROCHIP_TCPIP_V5` | Use Microchip TCP/IP stack version 5 |
| `MICROCHIP_TCPIP` | Use Microchip TCP/IP stack version 6 or later |
| `MICROCHIP_TCPIP_BSD_API` | Use BSD socket API with the Microchip TCP/IP stack (used alongside either TCP/IP stack define above) |
| `WOLFSSL_PIC32MZ_HASH` | Enable hardware-accelerated hashing (MD5, SHA, SHA-256) via the PIC32MZ Crypto Engine |
| `WOLFSSL_PIC32MZ_CRYPT` | Enable hardware-accelerated symmetric encryption (AES, 3DES) via the PIC32MZ Crypto Engine |
| `WOLFSSL_PIC32MZ_LARGE_HASH` | Enables large hashing support; requires exclusive application-layer access to the crypto hardware |

> **Note:** `WOLFSSL_PIC32MZ_LARGE_HASH` is defined automatically when `WOLFSSL_MICROCHIP_PIC32MZ` is active. The source comment states it "requires exclusive access to crypto hardware done at application layer."

### Configure Flags

No specific `./configure` flags are documented in the source material for this platform. Configuration is typically done via `user_settings.h` in MPLAB X IDE projects.

### IDE Project Files

wolfSSL ships MPLAB X project files for PIC24/XC16 under:

```
IDE/MPLABX16/
├── README.md
├── user_settings.h
├── main.c
├── wolfssl.X          # Builds the static wolfSSL library
└── wolfcrypt_test.X   # Runs wolfCrypt self-tests
```

**Important:** Import the `wolfssl.X` project into your MPLAB X workspace before building `wolfcrypt_test.X`. This ensures Makefiles are correctly generated.

The compiled static library is output to:
```
IDE/MPLABX16/wolfssl.X/dist/default/production/wolfssl.X.a
```

The `wolfcrypt_test.X` project depends on UART output code generated by Microchip's **MCC (MPLAB Code Configurator)** tool. See `IDE/MPLABX16/README.md` for the MCC UART generation steps.

### Enabling `user_settings.h`

wolfSSL should be configured to use a custom `user_settings.h` by defining:

```c
#define WOLFSSL_USER_SETTINGS
```

This is the standard approach for embedded builds where `./configure` is not used.

## 3. Platform-Specific Features

### Hardware Cryptography (PIC32MZ Crypto Engine)

The PIC32MZ Crypto Engine is supported through the header:

```
wolfssl/wolfcrypt/port/pic32/pic32mz-crypt.h
```

This file is active when `WOLFSSL_MICROCHIP_PIC32MZ` is defined. It includes Microchip-specific headers (`<xc.h>`, `<sys/endian.h>`, `<sys/kmem.h>`) and defines the hardware control structures used by the crypto engine.

**Hardware-accelerated hash algorithms** (enabled via `WOLFSSL_PIC32MZ_HASH`):
- MD5
- SHA-1
- SHA-256

If all three of `NO_MD5`, `NO_SHA`, and `NO_SHA256` are defined, `WOLFSSL_PIC32MZ_HASH` is automatically disabled.

**Hardware-accelerated symmetric ciphers** (enabled via `WOLFSSL_PIC32MZ_CRYPT`):
- AES
- 3DES (DES3)

If both `NO_AES` and `NO_DES3` are defined, `WOLFSSL_PIC32MZ_CRYPT` is automatically disabled.

### Networking

wolfSSL supports the following TCP/IP stacks on PIC32:

- **Microchip TCP/IP Stack v5:** Define `MICROCHIP_TCPIP_V5`
- **Microchip TCP/IP Stack v6+:** Define `MICROCHIP_TCPIP`
- **BSD Socket API:** Add `MICROCHIP_TCPIP_BSD_API` alongside either TCP/IP stack define
- **lwIP:** Define `WOLFSSL_LWIP`
- **FreeRTOS+TCP:** Define `FREERTOS_TCP`

If using a custom I/O layer (no TCP/IP stack integration), define `WOLFSSL_USER_IO`.

### Threading

PIC32 embedded builds are typically single-threaded. Define:

```c
#define SINGLE_THREADED
```

FreeRTOS is also a supported RTOS option; define `FREERTOS` if using it.

### Memory and Stack

- `WOLFSSL_SMALL_STACK` is recommended for memory-constrained PIC32 devices; it moves large stack buffers to the heap.
- `WOLFSSL_GENERAL_ALIGNMENT 4` is appropriate for 32-bit PIC32 targets.

## 4. Common Issues

### Hardware Crypto Mutual Exclusion
`WOLFSSL_PIC32MZ_LARGE_HASH` requires that the application exclusively manages access to the crypto hardware. wolfSSL does not internally arbitrate hardware access for large hash operations — this must be handled at the application layer.

### Older XC16 Compiler Compatibility
When using older versions of the XC16 compiler (for PIC24/PIC32 builds), `min` and `max` may not be defined. Add the following to `user_settings.h` if you encounter build errors:

```c
#define WOLFSSL_HAVE_MIN
#define WOLFSSL_HAVE_MAX
```

### No Filesystem
PIC32 embedded targets typically have no filesystem. Define:

```c
#define NO_FILESYSTEM
```

### No `/dev/random`
Hardware RNG must be provided separately. Define:

```c
#define NO_DEV_RANDOM
```

For testing only, `WOLFSSL_GENSEED_FORTEST` can be used, but this is **not suitable for production**.

### No `writev`
Standard POSIX `writev` is not available:

```c
#define NO_WRITEV
```

### Math Library Sizing
For PIC32 builds with RSA, `FP_MAX_BITS` should be set appropriately. The example configuration uses `2048` bits. Larger key sizes will require more RAM.

### MCC-Generated Files Required
The `wolfcrypt_test.X` MPLAB X project requires UART output code generated by MCC. The project will not build correctly without first generating these files.

## 5. Example Configuration

The following is a minimal `user_settings.h` for a PIC32MZ target with hardware crypto acceleration, based on the wolfSSL source material:

```c
#ifndef WOLFSSL_USER_SETTINGS_H
#define WOLFSSL_USER_SETTINGS_H

/* ---- Platform ---- */
#define WOLFSSL_GENERAL_ALIGNMENT   4
#define SINGLE_THREADED
#define WOLFSSL_SMALL_STACK

/* PIC32MZ platform + hardware crypto engine */
#define MICROCHIP_PIC32
#define WOLFSSL_MICROCHIP_PIC32MZ

/* Enable hardware-accelerated hash and symmetric crypto */
#define WOLFSSL_PIC32MZ_HASH
#define WOLFSSL_PIC32MZ_CRYPT

/* TCP/IP stack (choose one) */
/* #define MICROCHIP_TCPIP_V5 */
/* #define MICROCHIP_TCPIP    */
/* #define MICROCHIP_TCPIP_BSD_API */

/* ---- No standard OS features ---- */
#define NO_FILESYSTEM
#define NO_DEV_RANDOM
#define NO_WRITEV
#define WOLFSSL_USER_IO
#define BENCH_EMBEDDED

/* ---- Math ---- */
#define USE_FAST_MATH
#define FP_MAX_BITS     2048
#define TFM_TIMING_RESISTANT

/* ---- Crypto ---- */
#define HAVE_ECC
#define ECC_USER_CURVES
#define ECC_TIMING_RESISTANT

/* Disable unused algorithms to save space */
#define NO_DES3
#define NO_RC4
#define NO_MD4

/* ---- TLS ---- */
#define WOLFSSL_TLS13
#define NO_OLD_TLS

#ifdef __cplusplus
}
#endif

#endif /* WOLFSSL_USER_SETTINGS_H */
```

For a PIC24 / XC16 target, refer to the complete example at:
```
IDE/MPLABX16/user_settings.h
```
