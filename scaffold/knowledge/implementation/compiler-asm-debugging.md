# Compiler & Toolchain Assembly Debugging

> One-line summary: investigation patterns for inline assembly bugs across IAR/Keil/GCC/Clang — clobber lists, register binding, and the NO_VAR_ASSIGN_REG contract change.

**When to read**: Debugging optimization-dependent crashes in wolfSSL assembly, porting to a new compiler, or investigating inline asm correctness on ARM.

---

## Assembly Output Formats

wolfSSL generates assembly in three formats:
- `.S` (AT&T syntax) — GCC/Clang
- `.asm` (MASM) — MSVC
- `_c.c` (C inline assembly) — IAR/Keil/others via `WOLFSSL_ARMASM_INLINE`

All generated by Ruby scripts in the `wolfssl/scripts` repo. Read the first
40-50 lines of any `_c.c` file for the compiler detection preamble.

## GCC Inline Assembly Correctness Model

**Operand constraints** — classify every operand:
- `"r"` (input) — compiler picks register, asm reads it, compiler assumes unmodified
- `"+r"` (read-write) — same register for input AND output
- `"=r"` (output-only) — compiler assigns register for asm to write
- `"=&r"` (early-clobber) — output guaranteed different from all inputs

**Clobber contract**: lists registers modified but NOT operands. Compiler
avoids assigning operands to clobbered registers. `"memory"` = arbitrary
memory access. `"cc"` = condition codes modified.

## The Uncovered Register Check

For each hard-coded register write in the asm body (e.g., `MOV r4, ...`),
verify it is covered by EITHER an operand constraint OR the clobber list.

If a hard-coded register is in neither, the compiler may put a live value
there. The asm silently destroys it. **This is the root cause of most
optimization-dependent inline asm bugs.**

## Write-Before-Read Hazard

Asm preambles that save operands to scratch registers before computation:
```
MOV  lr, %[in]           // save %[in] to lr
MOV  r0, %[table_ptr]   // save table to r0 — but %[len] might BE r0
MOV  r12, %[len]         // reads %[len] — CORRUPTED if compiler put it in r0
```
The compiler is free to assign `%[len]` to r0 if r0 isn't clobbered.
Manifests at `-O2` when register allocator reorders assignments.

## WOLFSSL_NO_VAR_ASSIGN_REG

GCC's `register int x __asm__("r4")` forces variables into specific registers.
IAR/Keil don't support this syntax. When `WOLFSSL_NO_VAR_ASSIGN_REG` is
defined (automatic for IAR/Keil), the compiler freely assigns registers.

**The contract change**:
- **With binding**: register N = parameter N (guaranteed). Hard-coded register
  names in asm body are safe.
- **Without binding**: compiler assigns freely. Hard-coded names may collide
  with operand assignments.

**Bug pattern**: the Ruby generator's `asm_vars()` skips register N from the
clobber list when parameter N exists (correct for binding path, wrong for
no-binding path). Check the `_c.c` clobber lists for both `#ifdef` paths.

## ARM Register Budget

ARM has 13 GPRs (r0-r12; r13=SP, r14=LR, r15=PC reserved).
Rule: `operand_count + clobber_count <= ~12`.

If adding missing registers to the clobber list exceeds the budget, the fix
requires restructuring: splitting asm blocks, passing values via memory, or
reducing operand count. Thumb-2 narrow instructions further constrain to r0-r7.

## -O0 vs -O2: The Signature Symptom

At `-O0`, compilers assign operands in declaration order — often accidentally
matching the generator's assumptions. At `-O2`, liveness-based allocation
exposes latent register conflicts. If a bug is "works at -O0, breaks at -O2,"
check the clobber list first.

**`volatile` on data masks but does not fix clobber bugs** — it changes
allocation pressure, accidentally avoiding the conflict. Fragile; breaks
on any code/compiler change.

## Known Failure Modes

| Symptom | Root Cause | Key File/Function |
|---------|------------|-------------------|
| Crash at -O2 but works at -O0 | Missing register in clobber list | Uncovered register check on asm block |
| Wrong result on IAR/Keil only | `NO_VAR_ASSIGN_REG` clobber list gap | Compare clobber lists between `#ifdef` paths |
| Corrupt output from asm function | Write-before-read in preamble | Trace first N instructions for operand collision |
| Linker error on register constraint | Register budget exceeded | Count operands + clobbers vs 12 GPR limit |
| Oversized .bin from cross-compile | ELF FLASH-to-RAM gap in `objcopy` output | Check `readelf -Wl` for LOAD segment addresses |
